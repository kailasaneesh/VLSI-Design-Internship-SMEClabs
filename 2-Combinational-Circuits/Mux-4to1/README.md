A 4-to-1 Multiplexer (MUX) is a combinational logic circuit that acts as a digital data selector. It accepts four separate data input lines (i0, i1, i2, i3) and routes exactly one of them to a single output line (y). The selection is controlled by two control lines (s1, s0). The core rule is that the binary code on the select lines decides which specific data input gets passed through to the output.

In this simulation, the control lines are set to s1=1 and s0=0, which forms the binary code 10, representing the decimal number 2. This code tells the circuit to select data input line i2. Internally, the NOT gates invert the select lines to create s0_not=1 and s1_not=0. Looking at the logic equations, only the product term m2 gets a pure string of ones (i2 AND s1 AND s0_not = 1 AND 1 AND 1), which evaluates to 1. Since all four terms are combined by an OR gate at the end, the final output line y becomes 1.

The waveform graph visually verifies this selection logic during simulation. The input lines i0 through i3 are all driven to a high logic level, and the select lines show s1 sitting high while s0 sits flat. Consequently, the internal signal m2 steps upward along with the final output track y, which jumps into a high state (1). This clear graphical tracking confirms that the multiplexer successfully routes the correct target input channel to the output.
