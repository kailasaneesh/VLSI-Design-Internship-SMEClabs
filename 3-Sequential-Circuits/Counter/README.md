A 4-Bit Binary Counter is a sequential logic circuit that increments its output value by one on every active clock cycle. It uses an asynchronous reset signal (rst) and a clock signal (clk) to manage its state. The core rule is that the 4-bit output vector (q) tracks sequential binary values from 0000 up to 1111 (decimal 0 to 15). Once it reaches its maximum limit of 1111, the internal logic automatically resets the count back to 0000 on the next active clock edge, starting the cycle over again.

In this simulation, the reset line is kept inactive (rst=0) to allow normal operation. Internally, the code uses a process block that triggers with any change on the input lines. The logic monitors the clock line for a rising edge event. When a valid clock edge occurs, the circuit executes an addition step, adding one to the current tracking value (q <= q + 1). As seen in the code, a safety condition ensures that if the count attempts to exceed the maximum value of 1111, it is explicitly forced back to 0000.

This clear tracking pattern confirms the sequential counter operates smoothly without any timing errors.
